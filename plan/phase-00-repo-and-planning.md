# Phase 00 - Repo And Planning

## Objective

Prepare the repository and planning structure before writing application code.

## Agent Responsibilities

- Create the local project structure.
- Add `plan/` documents.
- Prepare a sensible root layout for backend, frontend, docs, and CI.
- Add initial `.gitignore` once implementation starts.
- Check that the remote GitHub repository is reachable.

## User Responsibilities

- Confirm the GitHub repository URL and whether it should be used directly.
- Decide whether the final database should be SQLite or PostgreSQL.
- Confirm the deadline and how much UI polish is expected.

## Test Strategy

No application tests yet.

Manual checks:

- Confirm the plan matches the assignment.
- Confirm the phase order feels realistic.
- Confirm no required feature is missing.

## Done Criteria

- `plan/` folder exists.
- Phase documents exist.
- Project direction is clear enough to begin architecture and database design.
