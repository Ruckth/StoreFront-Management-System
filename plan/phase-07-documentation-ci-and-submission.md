# Phase 07 - Documentation CI And Submission

## Objective

Prepare the final deliverables for submission.

## Required Deliverables

- Source code hosted on GitHub.
- ER diagram.
- Unit tests.
- README with setup instructions.
- Environment variable documentation.
- Brief architecture explanation.

## Optional But Valuable

- GitHub Actions workflow to run backend tests.
- Screenshots in README.
- Seed data command.
- Clear commit history.

## Agent Responsibilities

- Write README.
- Add `.env.example`.
- Add GitHub Actions workflow.
- Verify test command from clean instructions.
- Verify frontend build command.
- Check final repo status.
- Prepare submission summary.

## User Responsibilities

- Push final code to GitHub if credentials are local-only.
- Review README as if receiving the assignment.
- Submit repository link by email.

## README Should Include

- Project overview.
- Tech stack.
- Features.
- Architecture notes.
- ER diagram link or image.
- Backend setup.
- Frontend setup.
- Environment variables.
- Test commands.
- Demo user instructions if seed data exists.
- Known limitations or assumptions.

## Done Criteria

- README is complete.
- ER diagram is included.
- Tests pass.
- CI workflow exists if included.
- Repository is pushed.
- Submission email can be written confidently.
