# StoreFront Management System Plan

This folder breaks the assignment into phases that match two realities:

- Codex can build, inspect, run tests, debug, and verify many technical flows.
- You should manually test the business flow, UX clarity, and final submission quality.

Each phase includes:

- Objective
- Agent responsibilities
- User responsibilities
- Test strategy
- Done criteria

## Working Rule

After each phase, stop and verify before moving on. The assignment has a short deadline, so each phase should produce something reviewable, runnable, or testable.

## Suggested Definition Of Done

The project is ready to submit when:

- Backend API supports seller and buyer workflows.
- Frontend lets a real user complete the required stories.
- Role-based access is enforced.
- Checkout reduces inventory correctly.
- Unit tests cover important business rules.
- README explains setup, environment variables, architecture, and testing.
- ER diagram is included.
- GitHub repository has clean commits and a proper `.gitignore`.
